# 資料入力パイプラインに関する未策定事項

## 1. 本文書の目的

本書は、オーディオブック作成ツールにおいて、現時点では仕様が策定されていない資料入力関連の機能を整理するための仕様提案書である。

ここに記載する項目は、実装対象から恒久的に除外するものではない。

現在策定中のコンテンツ作成パイプラインおよび音声生成パイプラインとは分離し、今後「資料入力パイプライン」として別途検討・仕様化する予定の項目である。

本書に記載された内容は、要件、処理方式、入力形式、出力形式、エラー処理などが未確定の状態にある。

## 4. 想定する資料入力パイプライン

詳細は未策定であるが、現時点では次のような処理の流れを想定している。

```text
資料の登録
↓
ファイル形式の判定
↓
テキスト直接抽出または画像化
↓
必要に応じたOCR
↓
ページ単位の結果保存
↓
不要部分の除去
↓
文章と章構造の復元
↓
品質検査
↓
コンテンツ作成パイプラインへの受け渡し
```

この処理順序は暫定的なものであり、正式な仕様ではない。

ファイル形式や資料の状態によっては、異なる処理経路を使用する可能性がある。

例：

```text
テキスト埋め込みPDF
→ テキスト直接抽出
→ 前処理
```

```text
スキャンPDF
→ ページ画像化
→ OCR
→ 前処理
```

```text
EPUB
→ HTMLと目次の抽出
→ 前処理
```

```text
Kindle
→ 画面キャプチャ
→ OCR
→ 前処理
```

## 5. 他パイプラインとの境界

資料入力パイプラインの最終出力は、コンテンツ作成パイプラインが利用できる構造化された資料データを想定する。

ただし、具体的な受け渡し形式は未策定である。

暫定的には、次の情報を含むことを想定している。

* 資料ID
* 資料名
* 資料の種類
* 入手元
* ページまたは章の識別情報
* 抽出された本文
* 元ファイルとの対応情報
* OCRまたは抽出処理の種類
* 信頼度
* 警告
* 著作権および利用許諾に関する記録

想定例を次に示す。

```yaml
source_id: mysql_reference_manual
title: MySQL 8.0 Reference Manual
source_type: pdf
extraction_method: direct_text

sections:
  - section_id: section_001
    page_start: 1
    page_end: 3
    text: 抽出された本文
    confidence: 0.98
    warnings: []

rights:
  status: unverified
  usage_purpose: personal_learning
```

この形式は説明用の例であり、正式なスキーマではない。

## 6. 実装上の扱い

資料入力パイプラインの仕様が確定するまでは、関連機能を本体の必須要件として扱わない。

コンテンツ作成パイプラインは、資料入力パイプラインの完成を前提とせず、手動で用意されたテキストや構造化資料を入力として動作できるようにする。

```text
現段階の入力
＝ ユーザーまたは外部処理が用意した資料データ

将来の入力
＝ 資料入力パイプラインが生成した資料データ
```

この方針により、資料入力機能が未完成でも、原稿作成および音声生成機能の開発を進められるようにする。
