# AIによる追加機能提案と優先順位

## 目的

利用者が列挙した機能 (制作依頼作成・素材import・出力形式選択・声選択)以外に、
実際のオーディオブック作成運用を安全・便利にする機能をAIが提案し、
MVP、次期、将来へ分類する。

## 背景

`01-product-scope-and-mvp.md`のMVP/次期/将来分類を土台に、日常運用・品質確認・
エラー復旧・バックアップ・承認・履歴・コスト管理の観点から追加提案を行う
(利用者からの明示的な指示に基づく)。

## 対象

- 20案以上の追加機能候補。
- 価値・コスト・riskマトリクス。
- 採用推奨Top5。

## 対象外

- 各機能の詳細仕様化 (採択後に個別タスクとして扱う)。

## 既存仕様との関係

各提案は、既存承認済み仕様 (`07-approval-workflow.md`,`18-ai-model-routing-and-cost-control.md`等)
の既存機構をUI上に可視化・操作可能にするものが中心であり、新しい業務ルールを
勝手に追加するものではない。

## 用語

`00`の用語集を使用する。

## 採用推奨Top5 (次期最優先候補)

1. **承認待ち一覧 (#4)** — 低コストで`07-approval-workflow.md`の運用可視性を大きく改善する。
2. **AI利用コスト・使用量表示 (#10)** — `18-ai-model-routing-and-cost-control.md`の既存記録機構を
   活かすだけで実現でき、予算超過事故の予防効果が高い。
3. **バックアップ・復元のUI化 (#15)** — `13`の手動運用を減らし、誤操作対策込みで安全性を高める。
4. **Project archive/複製 (#1,#2)** — 運用上の基本機能でありコストも低い。
5. **Engine/API疎通診断 (#8)、定期ヘルスチェック (#25)** — MVPの「設定・診断」画面に統合し、
   エラー復旧の初動を早める。

## 不採用または保留理由

- 監修者コメントスレッド (#24): 単一利用者前提のMVPでは価値が低く、複数利用者対応 (将来区分)と
  合わせて再検討するのが合理的。
- 統合プレビュー (#5): 価値は高いが実装コストが高く、MVP完成を優先する`01`の方針と衝突するため
  将来区分とする。
- タグ・テンプレート (#13,#14): 便利機能だが、Project数が少ないうちは優先度が低い。
