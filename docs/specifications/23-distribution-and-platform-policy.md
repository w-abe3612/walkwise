# 配布・プラットフォーム方針

## 1. 目的

Electronデスクトップアプリとしての配布方針、対象プラットフォーム、
Python runtime等の同梱方針、ライセンス表示、更新方針、uninstall時の
データ扱いを定義する。

## 2. 対象範囲

- 初期対象プラットフォーム
- package/installer生成の将来方針
- Python runtime・ffmpeg等の同梱方法
- ライセンス一覧の同梱
- 自動更新の要否
- code signing未実施時の扱い
- 利用者データの保存場所とuninstall時の扱い
- 将来プラットフォーム

## 3. 対象外

- installer生成の実際の実行(本タスクでは行わない)
- 具体的な同梱手順のコード

## 4. 現行実装

現行コードにpackage/installer生成の仕組みは存在しない。本書は新規方針である。

## 5. 推奨仕様

### 5.1 初期対象プラットフォーム

初期対象はWindows x64のみとする。

### 5.2 package/installer

Electronのpackage/installer生成は将来実施する。本書策定時点では
installerを生成しない(生成作業は実装タスクとして別途行う)。

### 5.3 Python runtime・ffmpeg等の同梱

Python runtime、ffmpeg等の外部依存を配布物へどう同梱するか
(埋め込みPython配布、利用者環境のPythonを利用する方式等)は、
実装前のPoC(概念実証)で検証する事項とする。本書は同梱方針の選択肢を
実装時に評価するという方針までを確定し、具体的な同梱手段は確定しない。

### 5.4 ライセンス一覧

配布物には、Electron本体、利用した主要OSSライブラリ、TTSエンジン等の
ライセンス一覧を含める。

### 5.5 自動更新

初期MVPでは自動更新を必須にしない。更新は利用者が新しい配布物を
手動で入手し、再インストールする方式を初期方針とする。

### 5.6 code signing

初期配布ではcode signingを必須にしない。未署名のインストーラーは、
Windows上でSmartScreen等の警告が表示されるリスクがあることを記録する。
このリスクは既知のものとして受け入れ、配布方法の説明に警告への対処法を含める。

### 5.7 利用者データの保存

利用者データ(Projectデータ、SQLiteファイル)は、アプリのinstall先
ディレクトリへ保存しない(`17-local-data-persistence-policy.md` 5.7節参照)。

### 5.8 uninstall

アプリのuninstallは、アプリ本体(実行ファイル、install先ディレクトリの内容)を
削除するが、利用者データ領域は自動削除しない。利用者データの削除は、
別途明示的な操作(手動でのフォルダ削除、またはアプリ内の将来機能)による。

### 5.9 将来プラットフォーム

macOS、Linuxは将来候補とする。初期MVPでは対応しない。

## 6. 入力

- なし(本書は方針定義)

## 7. 出力

- 配布物(将来のpackage/installer生成タスクの成果物)

## 8. 必須項目

- 初期対象プラットフォーム(Windows x64)
- ライセンス一覧の同梱
- 利用者データをinstall先に保存しない方針

## 9. 任意項目

- code signing(初期は未実施)
- 自動更新(初期は非対応)

## 10. バリデーション

### Error

- 利用者データをアプリのinstall先ディレクトリへ保存する設計。
- uninstall時に利用者データを自動削除する設計。

### Warning

- code signing未実施のまま配布する場合のSmartScreen警告リスク。

## 11. 状態・エラー・警告

対象外(本書は配布方針の定義であり、実行時の状態遷移を持たない)。

## 12. 正常例

利用者がインストーラーを入手し、Windows x64環境へインストールする。
アプリ起動後、利用者データはinstall先とは別の標準的な利用者データ領域へ保存される。

## 13. 異常例

| 状況 | 扱い |
|---|---|
| 未署名インストーラーの実行時にSmartScreen警告が出る | 既知のリスクとして受け入れ、配布案内で対処法を説明する |
| uninstall時に利用者データも削除されることを利用者が期待する | 自動削除しない方針を配布案内で明示する |

## 14. テスト観点

- 利用者データがinstall先ディレクトリに存在しないことを確認する。
- uninstall後も利用者データ領域が残っていることを確認する。
- ライセンス一覧が配布物に含まれていることを確認する。

## 15. 移行・互換性

新規方針であり、移行対象となる既存配布物はない。

## 16. 未決定事項

なし。Python runtime・ffmpeg等の同梱方式は、実装前PoCで検証する事項として
本書が既定した方針(5.3節)の範囲内である。

## 17. 完了条件

- 初期対象プラットフォームが明示されている。
- package/installerの将来生成方針が明示されている。
- Python runtime等の同梱方法がPoC事項として明示されている。
- ライセンス一覧の同梱が明示されている。
- 自動更新を初期MVPで必須にしない方針が明示されている。
- code signing未実施時のリスクが明示されている。
- 利用者データの保存場所とuninstall時の扱いが明示されている。
- 将来プラットフォームが明示されている。
